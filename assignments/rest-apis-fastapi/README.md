# 📘 Assignment: Building REST APIs with FastAPI

## 🎯 Objective

Learn to build modern, production-ready REST APIs using the FastAPI framework. Create a fully functional API with proper routing, request validation, and documentation that follows industry best practices.

**Skills practiced:** HTTP methods, routing, request/response handling, data validation, API documentation

## 📝 Tasks

### 🛠️ Create a Basic API with FastAPI

#### Description

Build a foundational REST API using FastAPI that implements CRUD (Create, Read, Update, Delete) operations. Set up your project with proper structure, models, and endpoints to handle HTTP requests and responses.

#### Requirements

Completed API should:

- Define data models using Pydantic for request/response validation
- Implement GET, POST, PUT, and DELETE endpoints
- Use path parameters and query parameters appropriately
- Return proper HTTP status codes (200, 201, 404, etc.)
- Include error handling for invalid requests


### 🛠️ Add Advanced Features and Documentation

#### Description

Enhance your API with additional features and proper documentation. Implement relationships between data resources, add filtering/sorting capabilities, and generate automatic API documentation.

#### Requirements

Completed API should:

- Include automatic OpenAPI documentation (Swagger UI)
- Implement filtering and sorting for list endpoints
- Use request body validation with detailed field constraints
- Add proper docstrings and request/response descriptions
- Handle edge cases and validate all user input
